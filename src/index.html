ワークショップについて

このワークショップでは、GitHub Copilot Metrics ダッシュボードの設定と使用方法を学びます。

Copilot Metricsは、GitHubのCopilot機能の利用状況を分析し、開発チームの生産性向上を測定するためのツールです。

Copilot Metricsの概要

Copilot Metrics ダッシュボードは以下の機能を提供します：

  • チーム全体のCopilot使用率の可視化
  • コード補完の採用率とその効果の測定
  • 開発者の生産性向上の定量化
  • カスタムメトリクスの追加と分析

注意: このツールはGitHubの公式提供であり、組織内のCopilot利用状況を理解するのに役立ちます。

ダッシュボードのセットアップ

前提条件

  • Node.js 16.x以上
  • npm 8.x以上
  • GitHub Personal Access Token（読み取り権限付き）

インストール手順

git clone https://github.com/microsoft/copilot-metrics-dashboard.git
cd copilot-metrics-dashboard
npm install

設定ファイルの作成

cp .env.example .env
# .envファイルを編集してGitHub Personal Access Tokenを設定

警告: Personal Access Tokenは安全に保管し、リポジトリにコミットしないでください。

データの取得と表示

ダッシュボードを起動して、Copilotメトリクスの取得と表示を行います：

npm start

ブラウザが自動的に開き、http://localhost:3000でダッシュボードが表示されます。

データ収集の設定

データ収集の範囲とリポジトリの設定は以下の手順で行います：

  1. ダッシュボード上部の「設定」ボタンをクリック
  2. 対象となる組織とリポジトリを選択
  3. データ収集期間を設定
  4. 「適用」ボタンをクリックして設定を保存

ヒント: 初回のデータ収集には時間がかかる場合があります。GitHubのAPI制限に注意してください。

ダッシュボードのカスタマイズ方法

  • src/components/フォルダ内のコンポーネントをカスタマイズ

  • src/pages/内でページレイアウトを変更
